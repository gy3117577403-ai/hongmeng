# 部署后 15 分钟快速验收清单 v1.5.0

## 使用时机
每次重新部署、回滚或修改环境变量后，执行本清单。目标是在 15 分钟内确认核心流程可用。

## 1. 健康接口
访问：

```text
/api/health
```

确认返回 `ok:true`。

## 2. 登录
打开登录页，使用有效账号登录。确认进入工单资料库首页。

## 3. 新建工单
新建一个测试工单，填写工单号、产品名称、阶段、优先级、状态和进度。确认保存成功并出现在工单列表中。

## 4. 上传 PDF
选择测试工单和“原图”或“SOP指导书”分类，上传一个 PDF。确认上传成功并出现在文件列表中。

## 5. 上传图片
选择测试工单和“成品图”分类，上传 JPG 或 PNG。确认上传成功并显示缩略图。

## 6. 预览
分别预览 PDF 和图片。确认浏览器可打开预览链接。

## 7. 下载
下载单个文件。确认浏览器能获取文件。

## 8. 下载全部 ZIP
点击“下载全部”。确认 ZIP 能正常下载，且内部目录按资料分类组织。

## 9. 导入 CSV
打开系统设置，下载 CSV 模板，填写一个测试工单后导入。确认导入结果汇总正确。

## 10. 导出 CSV
在系统设置中导出工单 CSV 和文件清单 CSV。确认文件可下载，中文可读。

## 11. 操作日志
打开操作日志，确认能看到新建工单、上传、下载、ZIP 下载、导入或导出记录。

## 12. 修改密码
使用测试账号验证修改密码流程。修改成功后应要求重新登录。测试完成后按运维流程恢复测试账号状态。

## 13. PWA manifest
访问：

```text
/manifest.webmanifest
```

确认可以打开，且包含应用名称、图标和 standalone 显示模式。

## 14. 系统设置
打开系统设置，确认数据库状态、对象存储状态、最大上传大小和支持格式显示正常。

## 15. 退出登录
点击退出登录，确认返回登录页。

## 快速命令检查
本地或可访问环境可运行：

```bash
npm run smoke
```

该命令只检查 `/api/health`、`/manifest.webmanifest` 和 `/login`，不需要登录，也不包含任何密码。

## 通过标准
- 核心页面可访问。
- 登录、工单、上传、预览、下载可用。
- 数据导入导出可用。
- 健康接口正常。
- 操作日志可查看。
- 没有阻塞生产现场使用的问题。
