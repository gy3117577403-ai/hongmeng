# 缺料反馈与跟进模块

日期：2026-07-25

入口：`/workspace/procurement`

## 模块定位

本模块只处理仓库缺料反馈和后续跟进，不创建采购单号，不管理供应商、价格、
合同、付款或财务数据。

真实数据来源仍然是仓库配料任务：

1. 仓库在配料任务中登记“缺料”或“数量不足”；
2. 系统自动创建或续写同一条缺料跟进任务；
3. 负责人接收任务并持续更新进展、状态和预计解决时间；
4. 负责人可以把任务推进到“待仓库确认”；
5. 只有仓库确认异常解决后，任务才进入“已解决”。

仓库把异常类型改成非缺料异常时，现有跟进任务自动取消。以后再次登记缺料时，
系统复用同一任务并保留原有时间轴。

## 跟进状态

| 状态 | 含义 |
| --- | --- |
| 待接收 | 仓库刚提交缺料反馈，尚未接收 |
| 跟进中 | 已有人负责协调处理 |
| 等待物料 | 已有明确进展，等待物料或调拨结果 |
| 待仓库确认 | 跟进人员认为问题已经具备复核条件 |
| 已解决 | 仓库确认异常解决 |
| 已取消 | 仓库把异常调整为非缺料类型 |

“等待物料”必须填写预计解决时间。每次进度更新都必须填写跟进说明，并保留操作
人、时间、前后状态和负责人。

## 页面结构

- 顶部统计：全部反馈、待接收、跟进中、等待物料、待仓库确认和已解决。
- 左侧队列：按风险和预计时间展示缺料反馈。
- 中间工作台：显示工单、仓库反馈、负责人、预计时间、动态流程和进度表单。
- 右侧协同栏：显示风险提示、完整动态时间轴和仓库任务深链。

页面使用 CSS 透视、层叠阴影、渐变和 `transform` 构成轻量 3D 层次，不使用
WebGL 或重型动画依赖。系统启用 `prefers-reduced-motion` 时自动关闭非必要动效。

## 权限边界

- 管理员：查看、接收、分派和更新全部跟进任务。
- 班组长：查看并更新跟进任务。
- 普通员工：只读查看，不能更新。
- 仓库异常的最终解决仍遵循仓库管理原有权限和状态机。

缺料跟进不能修改生产数量、工艺路线、生产阶段、工时池或员工报工。

## 验收路径

1. 在仓库管理选择一张待配料任务。
2. 报告“缺料”或“数量不足”，填写说明和预计到料日期。
3. 保存后确认仓库页面显示缺料跟进状态。
4. 进入缺料跟进中心，确认新任务为“待接收”且没有采购单号字段。
5. 点击“接收任务”，任务进入“跟进中”并绑定当前账号。
6. 填写进展和预计时间，推进为“等待物料”。
7. 物料具备复核条件后推进为“待仓库确认”。
8. 返回仓库任务，填写解决说明并确认异常解决。
9. 回到缺料跟进中心，确认任务显示“已解决”，时间轴完整保留。

## 数据安全

- 所有更新使用任务版本号防止并发覆盖。
- 缺料反馈创建、状态更新和操作记录在同一数据库事务中完成。
- 模块不记录数据库连接、Token、对象存储密钥或任何外部采购系统凭据。
