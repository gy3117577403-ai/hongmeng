## 验收范围

- PDF / 图片统一缩放、旋转、平移和适应模式。
- PC 滚轮焦点缩放与双击切换。
- 平板 Pointer Events 捏合、单指拖动、单指双点和双指双点恢复。
- 当前页快捷目录、目录编辑 / 删除 / 排序和并发保护。

## 手势与预览结果

- 鼠标滚轮：通过；指针偏离中心缩放后页面内容按焦点位移，浏览器页面 `scrollY` 保持 0。
- 鼠标双击：通过；适应窗口与约 200% 间切换。
- 双指捏合：通过；Pointer Events 模拟从约 1 倍放大到 2 倍，停止后 PDF 清晰重绘。
- 单指拖动：通过；放大后可拖动，边界约束有效。
- 单指双点：通过；可放大并恢复。
- 双指双点：通过静态与 Pointer Events 路径检查；恢复适应窗口并保留旋转。
- PDF：只渲染当前页，旧 render task 会取消，附近页只预取元数据，DPR 上限为 2。
- 图片：旋转 90 度后重新计算适应比例；切换图片保留缩放 / 旋转并重新居中，未发生裁切。
- 全屏：通过；工具栏和关闭入口正常。

Android WebView 的既有 legacy PDF.js、Promise polyfill 和 ArrayBuffer 路径保持不变。本轮未连接 Android 真机，需在 APK / PWA 实机复验触摸手势和系统返回键，不将浏览器模拟结果冒充真机结论。

- 操作日志：四个目录 action 均已记录。
- API 响应检查：未包含 `objectKey`、S3 Secret 或签名 URL。

## 链接与阅读位置
