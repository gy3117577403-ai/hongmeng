# Windows 微盘导入助手协议排查

适用版本：v1.16.5.1-wecom-helper-protocol-fix

## 为什么不需要管理员权限

助手应用清单使用 `asInvoker`，只按当前普通 Windows 用户权限运行。浏览器协议写入 `HKEY_CURRENT_USER\Software\Classes\hongmeng-workorder-import`，不写 `HKEY_LOCAL_MACHINE`，因此不会弹出 UAC，也不需要管理员权限、PowerShell、`regsvr32` 或手工修改注册表。浏览器与助手应在同一个普通用户上下文运行；不要以管理员身份启动助手。

## 首次启动和协议自动修复

1. 从本项目成功的 GitHub Actions 下载 `hongmeng-workorder-wecom-import-helper-win-x64`。
2. 完整解压 ZIP，不能直接在压缩包内运行。
3. 建议放到 `%USERPROFILE%\Apps\工单资料库微盘导入助手\`，保留 ZIP 内全部运行时文件，不要只复制 EXE。
4. 普通双击 EXE。首次启动自动注册协议。
5. 如果助手文件夹后续移动到新目录，普通启动新位置的 EXE，助手会自动更新协议的图标和启动命令路径。

助手“连接诊断”显示“已注册 / 未注册 / 需要修复”。状态异常时点击“注册 / 修复浏览器协议”，成功提示“浏览器协议已注册，无需管理员权限”。

## 测试浏览器协议

在助手“连接诊断”点击“测试浏览器协议”。系统会打开安全自测 URI `hongmeng-workorder-import://ping`：

- 不创建工单导入任务。
- 不访问 S3，不上传文件。
- 助手已运行时，ping 由当前用户 Named Pipe 转发给现有窗口。
- 成功时显示“浏览器协议测试成功”。

## Chrome / Edge 确认弹窗

网页点击“重新唤起助手”后，浏览器出现“打开工单资料库微盘导入助手”提示时选择允许。网页不会整页跳走，并会继续轮询服务端任务状态。按钮短暂显示“正在唤起”，未连接时可以再次点击，但不要连续快速点击。

## 助手已运行但网页未连接

1. 查看助手顶部是否已显示当前客户、规格和分类。
2. 展开“连接诊断”，确认服务地址为正式站点、连接状态不是“连接失败”。
3. 点击“重新连接任务”。
4. 回网页查看任务状态；只有助手完成服务端 `connect` 握手后才显示“助手已连接”。
5. 仍失败时点击“复制诊断信息”交给维护人员。诊断内容经过脱敏，不包含 ticket、Cookie、密码、数据库连接串或对象存储密钥。

## 手动任务码备用连接

企业安全策略拦截自定义协议时：

1. 网页保留“从微盘导入”弹窗，找到 6 位手动任务码。
2. 普通双击助手。
3. 在助手顶部输入任务码并点击“连接”。
4. 成功后网页变为“助手已连接 / 等待文件”。

任务码与创建任务的用户、工单和分类绑定，只在任务有效期内使用一次；错误尝试有频率限制。助手不保存任务码，网页和服务端日志也不记录明文码。

## 任务过期

任务默认 10 分钟有效。过期后助手停止目录监控和后续上传，网页点击“重新创建任务”生成新任务和新任务码。旧任务码不能继续使用。

## 企业安全策略和 SmartScreen

当前便携包未使用正式代码签名证书。若 SmartScreen 提示未知发布者，仅在确认文件来自本项目成功的 GitHub Actions 后，根据公司规则选择“更多信息 -> 仍要运行”。如果企业策略禁止未知程序，请联系 IT 放行该 EXE。不要关闭 Windows Defender，也不要使用代码绕过 Mark-of-the-Web 或企业安全策略。

## 更新助手

助手“检查更新”会打开 GitHub Actions 构建页。下载最新成功 run 的 artifact，退出旧助手，用新 ZIP 中的 EXE 替换固定目录文件并普通启动；协议路径会自动校验和修复。
