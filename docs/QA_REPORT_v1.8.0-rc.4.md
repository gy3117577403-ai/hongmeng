# QA Report v1.8.0-rc.4

- 版本：v1.8.0-rc.4

- 状态：本地验证通过，等待统一部署

## 变更说明
- 是否新增 migration：否
- 是否新增依赖：否
- 是否新增 API：否
- 本次只优化主页布局，不改上传、下载、S3、PostgreSQL、登录、账号、工单、导入导出等业务逻辑。

## 主预览区优先布局验收
- 已取消主区域固定分类 / 操作横条，当前分类由左侧分类栏表达，资料操作进入右侧工具窗。
- 已取消固定文件标题区，文件信息改为预览区左上角悬浮胶囊。
- 主预览区使用剩余可用高度，PDF / 图片预览区域不再被固定标题和操作横条挤压。
- Toast 调整到 AppBar 下方顶部居中，不覆盖右侧工具窗、缩略条和文件操作区。

## 右侧覆盖式工具窗验收
- 默认仅显示右侧窄工具条。
- 信息 / 上传 / 操作 / 队列点击后从右侧覆盖展开，不改变主预览区实际宽度。
- 工具窗支持拖拽宽度，范围 260px - 420px。
- localStorage 仅保存展开状态、当前 tab 和工具窗宽度，不保存敏感数据。

## 缩略图验收
- 底部缩略图默认收起，仅显示“文件 N 个”胶囊。
- 展开后为透明 glass 浮层，宽度受限，不参与布局高度计算。
- 选中文件使用橙色边框，图片缩略图、PDF 卡片、版本和最新 / 历史状态正常显示。

## PDF / 图片预览回归
- PDF.js 预览仍使用同源 canvas 渲染，无浏览器原生黑色工具栏。
- PDF 翻页、缩放、适宽、整页和全屏通过。
- 图片使用 `object-fit: contain` 完整显示，缩放和全屏通过。
- 中文文件名正常显示，未出现 URL 编码乱码。

## 1024x768 验收
- 1024x768 下无横向溢出。
- 当前工单条、左侧分类栏、右侧工具条和主预览区均可用。
- 工具窗展开为覆盖式，不挤压 PDF / 图片预览。

- 暂无阻塞问题。
