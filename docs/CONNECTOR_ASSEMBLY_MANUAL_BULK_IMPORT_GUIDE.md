# 连接器组装说明书批量导入指南

版本：v1.16.2-connector-manual-workspace-polish

状态：本地验证候选，尚未部署 Sealos。

## 为什么优先使用批量导入

批量导入面向几十到数百份 PDF 或图片说明书。用户先选择文件夹或多个文件，系统以文件名生成默认名称，完成轻量解析和数据库去重后再确认上传，无需逐份填写完整表单。原“单份新增”仍保留，适合少量资料、需要在上传前精细维护字段的场景。

## 文件选择与默认名称

页面顶部“批量导入说明书”是主入口，支持：

- 选择一个本地文件夹。
- 一次选择多个 PDF、JPG、JPEG、PNG 或 WEBP 文件。
- 自动忽略空文件、隐藏路径、非支持格式、`.part`、`.crdownload` 和 `~$` 临时文件。

PDF 每个文件默认是一份说明书的一个版本。说明书默认名称始终是原文件名去掉扩展名；封面检测标题单独保存为 `detectedTitle`，不会强制覆盖默认名称。用户可在预览表中修改名称。

## 推荐文件夹结构

```text
说明书根目录/
  制造商A/
    系列A/
      型号A-组装说明书-Rev01.pdf
      型号A-组装说明书-Rev02.pdf
  制造商B/
    系列B/
      图片说明书名称/
        01.jpg
        02.jpg
```

一级父目录作为制造商候选，二级父目录作为系列候选，只提供建议，不会覆盖用户确认的数据。组装说明、组装说明书、操作说明、产品说明、目录、说明书和装配说明属于通用词，不会写成正式制造商。

## 图片集规则

- 同一子文件夹中的图片按文件名自然排序，并组成一份图片集说明书。
- 直接位于所选根目录的图片各自作为单页说明书。
- 预览前可勾选图片候选后执行“合并所选图片”或“拆分所选图片集”。
- 一个图片集作为一个队列任务上传，内部图片顺序写入 `pageNo` 和 `sortOrder`。

## 自动识别

PDF 预览阶段复用现有 `pdfjs-dist`，并发最多为 2，只读取页数、metadata、第 1 页文本和可选第 2 页文本，不渲染全部页面。系统尝试识别：

- 封面标题。
- 制造商和连接器系列候选。
- `Rev 01`、`REV A`、`V1.0`、`R01`、`2024版` 等版本。
- 标准日期。
- 文件名或正文中重复出现的连接器型号。
- 带页码的章节名称和关键词。

识别失败只会产生警告，仍可按文件名导入。预览文本不写入 `localStorage`，也不在浏览器控制台打印 PDF 全文。

解析元数据同时标记 `confirmed`、`detected`、`needs_review`。文件名主名称为 `confirmed`，封面检测标题只作为 `detectedTitle` 建议；无法确认的制造商保持空并进入待完善，不会用低置信度结果覆盖用户输入。

## 预览、去重和版本规则

预览 API 只接收元数据与 SHA-256，不上传文件、不写数据库、不创建说明书，也不返回签名链接或对象存储凭据。每条候选会分类为：

- `create_manual`：新建说明书和首版。
- `create_version`：同名说明书的新版本。
- `duplicate`：有效资产中已有相同 SHA-256，默认跳过。
- `conflict`：名称和版本相同，但文件内容不同。
- `invalid`：文件为空、类型不支持或元数据不完整。
- `manual_review`：无法安全判断新建或新增版本。

冲突项必须改成新版本号或跳过。系统不会覆盖旧 S3 对象；新增版本总是使用新的 object key。

## 参数关联建议

解析出的型号只与现有连接器参数做精确匹配：

- 唯一精确匹配可默认勾选关联。
- 多条精确匹配只显示为待确认建议。
- 无匹配不关联，也不阻塞导入。

批量设置可统一补充制造商、系列、关键词，并控制是否自动关联唯一精确匹配。系统不会通过模糊搜索批量绑定参数。

## 确认与上传队列

预览确认后必须输入 `IMPORT_MANUALS`。系统先创建持久化导入批次，再按一个 PDF 或一个图片集一个请求上传，前端并发为 2。同名说明书任务会串行执行，避免首版和新版本竞争创建。

队列支持：

- 暂停和继续未开始任务。
- 取消尚未开始的任务。
- 单项失败不影响其他文件。
- 重试失败项。
- 导出失败清单 CSV。
- 查看最近导入批次和每项结果。

`batchId + clientId` 形成唯一任务，服务端还会重新计算实际上传内容的 SHA-256。重复提交已成功项会返回幂等结果，不会再次创建说明书或版本。

## 上传后处理

文件成功写入 S3 兼容对象存储后，系统在 PostgreSQL 创建或更新说明书、版本、资产和精确匹配关联，并写入操作日志。PDF 正文在服务端提取后进入搜索索引。全文提取失败不会回滚已上传文件，而是把版本标记为 `partial` 或 `failed`，仍可按文件名、默认名称、相对路径等检索。

搜索索引覆盖原始文件名、默认名称、检测标题、相对路径、制造商、系列、版本、型号、关键词、章节和可提取的 PDF 正文。

## 待完善筛选

说明书列表常驻“全部、最新版、待完善、未关联、解析失败”快捷筛选；制造商、系列、型号与发布日期范围位于筛选浮层。首次导入不要求资料完全完善；制造商未确认、版本未识别、发布日期未识别、型号未关联或解析失败的记录可后续补齐。

## 单份新增

“单份新增”是次级入口，改为选择文件、自动解析、确认保存三步。初始只要求文件和名称，高级信息默认折叠；只有点击“使用识别结果”才补充空字段，已经输入的名称或制造商不会被覆盖。保存时直接完成文件上传。

## 数据安全边界

- 所有账号继续共享同一套业务数据，不引入角色权限。
- 文件只进入 S3 兼容对象存储，不长期保存在容器本地。
- 元数据和导入批次保存在 PostgreSQL。
- 说明书、版本和资产删除仍是软删除，不物理删除 S3 文件。
- 批量导入不删除连接器参数，不覆盖已有历史版本。
- 文档、日志和 API 不应包含密码、Token、数据库连接串或对象存储密钥。
