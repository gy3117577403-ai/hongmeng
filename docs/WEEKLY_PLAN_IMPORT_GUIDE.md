# 周计划 Excel 工单导入说明

版本：v1.13.5-weekly-plan-import

## 支持格式

- `.xls`
- `.xlsx`
- `.csv`

导入入口在系统设置的“工单批量导入”区域。周计划文件上传后会先进入预览，不会直接写入数据库。

## 支持字段

| 周计划表头 | 系统字段 | 说明 |
| --- | --- | --- |
| 订单日期 | `orderDate` | 可识别 `2026/06/03` 等日期 |
| 业务员 | `salesperson` | 特殊 SO 错位行会改写 |
| 客户名称 | `customerName` | 工单客户 |
| 客户等级 | `customerLevel` | 用于优先级映射 |
| 品名 | `productName` | 工单产品名称；为空但规格存在时用规格补齐 |
| 规格 | `specification` | 工单规格 |
| 工序 | `processName` | 生产工序 |
| 未交量 | `uncompletedQty` | 原样保存为文本 |
| 工时 | `unitWorkHours` | 原样保存为文本 |
| 总工时 | `totalWorkHours` | 原样保存为文本 |
| 图纸 | `drawingStatus` | 原样保存 |
| 交期 | `deliveryDay` + `plannedAt` | 保存周几文本，并按计划周开始日期推算具体日期 |
| 配料 | `materialStatus` | 原样保存 |
| 备注 | `remark` | 原样保存 |
| 图纸下发日期 | `drawingIssuedAt` 或 `drawingIssueNote` | 可识别日期则保存日期，否则保存说明文本 |

## SO 单号错位行

如果“业务员”列以 `SO` 开头，例如：

```text
业务员列：SO20260603003
客户名称列：李苗苗
客户等级列：杭州云深处(10126)
```

系统按以下规则修正：

- `sourceOrderNo = SO20260603003`
- `salesperson = 李苗苗`
- `customerName = 杭州云深处(10126)`
- `customerLevel = null`

这样不会把来源订单号误当业务员，也不会把业务员误当客户。

## 合计行和空行跳过

以下行会在预览中显示为“跳过”：

- 整行为空。
- 只有总工时或合计说明，没有品名和规格。
- 包含“合计工时”“本周+上周计划滚动合计工时”等说明。
- 没有品名且没有规格的行。

跳过行不会写入数据库。

## 交期日期映射

周计划导入模式会显示“计划周开始日期”。例如填写：

```text
2026-07-06
```

则映射为：

- 周一 -> 2026-07-06
- 周二 -> 2026-07-07
- 周三 -> 2026-07-08
- 周四 -> 2026-07-09
- 周五 -> 2026-07-10
- 周六 -> 2026-07-11
- 周日 -> 2026-07-12

如果未设置计划周开始日期，系统仍保存 `deliveryDay`，但 `plannedAt` 为空，预览中会提示。

## 图纸 / 配料 / 状态映射

`drawingStatus` 和 `materialStatus` 会原样保存，同时用于初始化工单状态：

- 图纸为空 -> `未发图`
- 图纸为“已发”，配料包含“已配料”或“料齐” -> `在后端`
- 图纸为“已发”，但配料不齐 -> `在前端`

优先级映射：

- 客户等级 `A` -> `高`
- 客户等级 `B` -> `一般`
- 备注包含“新品验证” -> `一般`
- 其他默认 `一般`

后续如果用户手动修改状态或优先级，系统保留用户修改。

## 预览和确认

导入流程：

1. 选择“周计划 Excel 导入”。
2. 填写或确认计划周开始日期。
3. 上传 `.xls` / `.xlsx` / `.csv`。
4. 查看预览表。
5. 确认可导入、重复、跳过、异常行。
6. 点击“确认导入”。

预览会返回：

- 总行数
- 可导入行
- 重复行
- 跳过行
- 异常行
- 每行解析后的工单号、客户、品名、规格、未交量、图纸、配料、交期、计划日期和结果

重复行默认跳过，也可以选择“仍然导入重复行”。系统会保证最终工单号唯一。

## 常见错误

- 文件格式不支持：请上传 `.xls`、`.xlsx` 或 `.csv`。
- 表头缺失：确认第一张表中包含“订单日期、业务员、客户名称、品名、规格”等字段。
- Excel 文件读取失败：请重新另存为 `.xlsx` 后再试。
- 日期无法解析：检查订单日期或交期日期格式。
- 没有可导入行：确认不是合计表、空表或只有说明文字。

## 数据安全

周计划导入只写入工单数据，不会修改：

- 连接器参数
- 连接器参数附件
- 连接器导入批次
- S3 对象存储配置
- 登录账号和密码

本文档不包含任何数据库密码、S3 Key、SESSION_SECRET、admin 密码或真实生产密码。

## v1.13.6 显示和清理补充

从 v1.13.6 开始，周计划导入工单的主显示编号使用“规格”：

```text
displayCode = specification || code
```

系统内部仍会生成并保留唯一 `code`，用于数据库关联、导入去重和安全确认。现场页面、工单卡片、当前工单信息条、ZIP 下载文件名和打印摘要优先显示规格 / 生产编号。

周计划导入写入：

- `planType = weekly_plan`
- `planActive = true`
- `libraryKey = specification`
- `drawingLibraryItemId = 已存在且匹配的 DrawingLibraryItem.id`，如果没有匹配记录则为空
- `weekStartDate = 导入时选择的计划周开始日期`
- `weekEndDate = weekStartDate + 6 天`

同时，系统会按 `customerName + "::" + specification` 查找已有长期图纸资料库记录。如果客户为空，则按 `specification` 匹配。找到则关联；找不到不再自动创建空图纸资料记录。图纸资料库只保存客户、客户编码、规格、品名、备注和资料文件，不保存图纸状态、配料状态、交期、未交量、工时、业务员、订单日期等周计划字段。

如需开始新一周生产，可在系统设置中使用“本周生产工单清理”：

1. 选择计划周开始日期。
2. 点击“预览清理”。
3. 确认不会删除图纸资料库、资料文件和连接器参数。
4. 输入 `CLEAR_WEEK` 后执行。

清理只会把本周周计划工单设为 `planActive=false`，不会删除 `WorkOrder`、`DrawingLibraryItem`、`DrawingLibraryFile`、`ResourceFile`、S3 文件、连接器参数或连接器附件。
