# 企业微信微盘本地导入助手使用指南

版本：v1.16.5.4-wecom-helper-dragdrop-intake-fix

## 用途与边界

Windows 导入助手用于缩短“微盘下载 -> 网页找文件 -> 上传”的人工链路。它不接入企业微信 API，不需要 CorpID、AgentID、Secret 或管理员授权，也不会抓取企业微信 Cookie、私有缓存或自动操作企业微信界面。用户仍需在微盘中主动拖出、复制或下载文件。

## 下载和首次运行

1. 打开 GitHub 仓库的 `Actions` 页面。
2. 选择 `Windows WeCom Import Helper` 的成功构建。
3. 下载 artifact `hongmeng-workorder-wecom-import-helper-win-x64`。
4. 完整解压 ZIP，建议放到 `%USERPROFILE%\Apps\工单资料库微盘导入助手\`，不要直接在压缩包中运行，也不要只复制其中的 EXE。
5. 普通双击 `Hongmeng.WorkOrder.ImportHelper.exe`。首次运行会在当前 Windows 用户范围自动注册 `hongmeng-workorder-import://`。
6. Artifact 自带 .NET 运行时，启动前不做单文件运行时自解压；不需要管理员权限、PowerShell、手工修改注册表或重启 Windows，也不要“以管理员身份运行”。

当前构建未使用代码签名证书，Windows 可能显示“未知发布者”。请遵循公司终端安全策略，不要从非官方仓库来源下载助手。

## 从网页创建任务

1. 登录工单资料库并进入目标工单。
2. 选择目标资料分类。
3. 在右侧“上传”工具窗点击“从微盘导入”。
4. 网页创建 10 分钟短期任务，并尝试唤起助手。
5. 浏览器询问是否打开外部应用时选择允许。

如果浏览器协议被企业策略阻止，网页会同时显示 6 位一次性“手动连接码”。普通双击助手，在顶部输入任务码并点击“连接”。任务码与原用户、工单和分类绑定，10 分钟内只允许一个助手实例绑定；同一助手的双击或网络重试会幂等恢复任务，不需要输入账号密码。

任务只允许上传到创建时绑定的工单和分类，默认最多 20 个文件，并复用系统现有单文件及总大小限制。过期后助手立即停止监控和继续上传。

## 三种接收方式

### 拖拽

从微盘窗口拖出一个或多个真实文件到助手。若微盘只提供文本或链接，助手不会抓取链接，会提示改用下载目录监控。

### 复制粘贴

在微盘中复制一个或多个文件，切换到助手按 `Ctrl+V`。纯文本、链接和剪贴板位图不会被自动保存或上传。

### 下载目录监控

1. 点击“选择目录”，选中企业微信微盘下载目录。
2. 点击“开始监控”。
3. 在微盘点击下载。
4. 助手只处理本任务启动后新增的文件。

`.tmp`、`.part`、`.crdownload`、隐藏文件、0 字节文件会被忽略。文件需要连续 3 秒大小稳定并可只读打开，避免上传半成品。目录路径只保存在当前 Windows 用户本机，不上传服务器。

## 校验、重复和版本

- 支持 PDF、JPG、JPEG、PNG、WEBP。
- 校验扩展名、文件头、大小、图片解码及 SHA-256。
- PDF 原文件直接上传，不拆页、不转图、不压缩重编码。
- 同工单、同分类、相同 SHA-256：显示“完全重复”并默认跳过。
- 同名但 SHA-256 不同：显示“新版本”，保留旧文件并上传新记录。
- 同名同大小且旧文件哈希暂时不可确认：显示“需要确认”，用户明确勾选后才允许继续。

上传成功后，服务端写入 S3 兼容对象存储、创建 `ResourceFile`，并复用同一个 objectKey 同步图纸资料库，不重复上传对象。助手从不接触 S3 凭据。

## 1.16.5.4 文件接收

- Explorer 本地文件：拖过橙色区域时显示复制效果，松开后异步进入现有校验队列。
- Shell 虚拟文件：支持 `FileGroupDescriptorW/FileContents`，按文件索引流式落入当前任务临时目录；不读取企业微信 Cookie、私有缓存或 URL。
- 文件剪贴板：`Ctrl+V` 与拖放复用同一接收和安全校验流程。
- 仅链接：不自动抓取，提示改用下载目录监控。
- 拖放诊断：只显示 OLE 格式名称和类型判断，不显示文件正文、任务码、票据或下载地址。
- 下载目录：首次选定后自动复用，任务连接即开始监控；可选择自动上传或仅加入队列。

虚拟文件名会移除路径段、非法字符和 Windows 保留设备名；单文件、文件数和总大小都受当前短期任务限制。临时文件在失败、重复、上传成功、任务过期、任务移除或助手退出时清理。本地原文件和下载目录文件不会被删除。

## 队列操作

- `上传全部`：并发 2 个文件。
- `暂停队列 / 继续队列`：暂停或继续启动后续文件；正在传输的单个文件先完成。
- `重试失败`：只重试失败项，不重建已成功记录。
- `清空已完成`：只清理助手内存列表，不删除本地文件或线上资料。
- `移除任务`：清除助手内存中的任务和票据，不删除任何文件。

## 故障排查

- 助手未连接：允许浏览器打开外部应用；回到网页点击“重新唤起助手”，或普通双击助手并输入手动任务码。
- 协议未注册 / 需要修复：展开助手“连接诊断”，点击“注册 / 修复浏览器协议”。协议写入当前用户 HKCU，不需要管理员权限。
- 测试协议：点击“测试浏览器协议”；助手收到 `ping` 后显示“浏览器协议测试成功”，不会创建上传任务。
- EXE 移动目录：普通启动新路径下的 EXE，会自动修复协议命令路径。
- 任务过期：回网页重新点击“从微盘导入”。
- 没收到拖拽文件：微盘可能只提供链接，使用下载目录监控。
- 文件一直等待：确认下载已完成，且其他程序不再占用文件。
- 401 / 403：任务授权已失效，重新创建任务。
- 上传失败：检查网络后点击“重试失败”；本地文件仍原样保留。
- 需要反馈：在“连接诊断”点击“复制诊断信息”；报告不含 ticket、Cookie、密码或对象存储密钥。

完整排查流程见 [Windows 微盘导入助手协议排查](WECOM_HELPER_PROTOCOL_TROUBLESHOOTING.md)。

本文档不包含账号密码、票据、数据库连接串、对象存储密钥或任何真实生产文件。
